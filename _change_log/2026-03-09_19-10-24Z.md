## Problem
The GLM API caller was using the OpenAI-compatible endpoint (`/api/paas/v4/chat/completions`) which returned HTTP 429 "Insufficient balance" error.

## Root Cause
The user's Z.AI GLM subscription works with the Anthropic-compatible endpoint (`/api/anthropic`), not the OpenAI-compatible endpoint.\n\n## Solution\n- Changed endpoint from `https://api.z.ai/api/paas/v4/chat/completions` to `https://api.z.ai/api/anthropic`\n- Changed model from `glm-5` to `claude-3-5-sonnet-20241022`\n- Changed auth from `Authorization: Bearer` to `x-api-key` header\n- Added `anthropic-version: 2023-06-01` header\n- Added `max_tokens: 4096` parameter to requests\n- Updated response parsing from OpenAI format to Anthropic format\n\n## Files Modified\n- `app/src/main/kotlin/com/glassthought/Constants.kt`\n- `app/src/main/kotlin/com/glassthought/directLLMApi/glm/GLMHighestTierApi.kt`\n- `app/src/test/kotlin/com/glassthought/directLLMApi/glm/GLMHighestTierApiTest.kt`\n\n## Verification\nAll 14 unit tests pass.
