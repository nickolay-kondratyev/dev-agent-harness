## What changed

Removed question deduplication (ap.Girgb4gaq2aecYTHjUj8a.E) from all spec docs.

**Rationale**: Asking the human the same question twice is harmless — they answer again and the agent continues. Cost of ~30s in a rare edge case is cheaper than the complexity: pendingQuestions map, dedup lookup, answer routing, extra state to manage and test.

**Files updated (spec only, per task instructions)**:
- `doc/core/UserQuestionHandler.md`: removed entire dedup section, simplified flow from 11 to 10 steps
- `doc/core/SessionsState.md`: removed `pendingQuestions` field from SessionEntry table
- `doc/core/agent-to-server-communication-protocol.md`: removed dedup note from user-question endpoint
- `doc/high-level.md`: removed dedup note from Q&A mode row
- `doc/core/PartExecutor.md`: removed dedup AP ref from user-question side-channel row
- `doc/core/AgentInteraction.md`: updated R2 heading
