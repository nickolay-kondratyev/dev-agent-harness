## What Changed

All V1 specs simplified to remove EMERGENCY_INTERRUPT (hard threshold, Ctrl+C interrupt, 1-second context polling, race guards, auto-compaction disable config).

### V1 Approach (simplified)
- Single compaction trigger: `DONE_BOUNDARY` (35% context remaining, checked at done boundaries)
- Claude Code native auto-compaction remains ENABLED as safety net for mid-task exhaustion
- Health-aware await loop: no continuous context window polling, only standard health checks

### V2 (preserved in doc_v2/)
- Custom emergency compression with Ctrl+C interrupt at 20% threshold
- Auto-compaction disabled (harness-controlled only)
- Full design captured in `doc_v2/our-own-emergency-compression.md`
