## Changes

- Added `val model: String` as 3rd field to `ResumableAgentSessionId` data class
- Added `model: String` constructor parameter to both primary and test-injection constructors of `ClaudeCodeAgentSessionIdResolver`; stored as `private val model`
- Refactored `ClaudeCodeAgentStarterBundleFactory.create()` to extract `val model` (instead of inline per branch) and pass it to both `ClaudeCodeAgentStarter` and the new `ClaudeCodeAgentSessionIdResolver` constructor
- Updated `AgentSessionIdResolver` interface KDoc to mention model in `@return`
- Updated 12 unit test instantiations and all assertions in `ClaudeCodeAgentSessionIdResolverTest`
- Tightened integration test model assertion to `shouldBe "sonnet"` (was `shouldNotBeBlank()`)

## Architecture

`AgentSessionIdResolver` interface unchanged — model is injected at construction time via the factory, not passed to `resolveSessionId()`.

## Why

Claude Code cannot switch models when resuming a session. The model field in `ResumableAgentSessionId` enables future `ResumeTmuxAgentSessionUseCase` (ticket `nid_d47u5pku4ldixx23tyggd29ep_E`) to build:
`claude --resume <sessionId> --model <model>`
