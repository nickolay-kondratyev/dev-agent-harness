## TicketShepherd Implementation

### New Files
- `TicketShepherd.kt` — central coordinator with `TicketShepherdDeps` bundle
- `PartExecutorFactory.kt` — fun interface for testable executor creation
- `FinalCommitUseCase.kt` — fun interface for final git commit
- `TicketStatusUpdater.kt` — fun interface for marking ticket done
- `TicketShepherdTest.kt` — 22 BDD test cases

### Modified Files
- `ConsoleOutput.kt` — added `printlnGreen`, changed from `fun interface` to `interface`
- `ShepherdValType.kt` — added `PART_NAME`
- 3 test files updated for `printlnGreen` on FakeConsoleOutput

### Design Decisions
- `TicketShepherdDeps` bundles 11 collaborators to satisfy detekt's LongParameterList
- `PartExecutorFactory` decouples TicketShepherd from PartExecutorImpl construction
- `run()` returns `Nothing` since both paths exit the process
- Real implementations for factory/commit/updater are follow-up work
