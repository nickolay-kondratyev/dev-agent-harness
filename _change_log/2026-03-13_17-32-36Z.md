## Motivation
The `/validate-plan` endpoint broke the fire-and-forget pattern by returning a meaningful response body. Rather than eliminating it, we formalized the two communication patterns as explicit tiers.

## Changes
- **New two-tier design** in `doc/core/agent-to-server-communication-protocol.md`:
  - Tier 1 — Signals (`/callback-shepherd/signal/*`): fire-and-forget, bare 200 OK
  - Tier 2 — Queries (`/callback-shepherd/query/*`): synchronous request/response with meaningful body
- **Consolidated 6 scripts into 2**: `callback_shepherd.signal.sh <action>` and `callback_shepherd.query.sh <action>`
- **Updated all referencing files**: high-level.md, ContextForAgentProvider.md, PartExecutor.md, SessionsState.md, TicketShepherd.md, git.md, UserQuestionHandler.md, HealthMonitoring.md, SpawnTmuxAgentSessionUseCase.md, DetailedPlanningUseCase.md, AutoRecoveryByAgentUseCase.md, + doc_v2 files

## Robustness Benefit
- New endpoints land in the right tier by default — the naming makes the contract self-documenting
- Server-side routing is cleanly separated: signal handlers manage deferreds/timestamps, query handlers return data
