## Architecture

4 files in `core/context/` package:

1. **ProtocolVocabulary.kt** — Central registry of all protocol keywords (feedback statuses, severities, done results, signals). Single source of truth referenced by both instruction text and tests.

2. **InstructionSections.kt** — All static text sections as named vals/functions. Each section is independently searchable and editable. Uses string templates referencing ProtocolVocabulary for compile-time keyword linking.

3. **ContextForAgentProvider.kt** — Interface with 3 methods (execution agent, planner, plan reviewer) + request data classes. Factory pattern.

4. **ContextForAgentProviderImpl.kt** — Assembly logic using `buildList` for readable linear concatenation order. Per-section private methods with descriptive names.

## Testing

- **Keyword presence tests** (3 test classes): Verify protocol keywords are present in assembled instructions via `shouldContain` — not brittle exact-match.
- **Structural tests**: Verify conditional section inclusion (PLAN.md for with-planning only, feedback state for iter > 1, pushback guidance for doer iter > 1).

## Key Design Decisions

- Static text in Kotlin constants (not resource files) — compile-time link to ProtocolVocabulary
- buildList shows concatenation order as readable linear sequence (no hidden abstraction)
- V1 recompile acceptable for tweaks — sections easy to find via IDE symbol search
