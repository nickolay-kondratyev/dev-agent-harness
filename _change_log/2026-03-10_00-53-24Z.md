## Workflow JSON Parser

### What
- `WorkflowDefinition` domain model with nullable fields for straightforward vs with-planning differentiation
- `WorkflowParser` interface + `WorkflowParserImpl` using Jackson + Kotlin module
- Post-deserialization validation: blank name, mutual exclusivity, empty phases, incomplete planning config
- Production workflow files: `config/workflows/straightforward.json`, `config/workflows/with-planning.json`
- 32 BDD unit tests covering happy paths and error paths

### Dependencies Added
- `jackson-databind:2.17.2`
- `jackson-module-kotlin:2.17.2`
