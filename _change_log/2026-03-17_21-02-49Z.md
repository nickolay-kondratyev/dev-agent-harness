## What changed

Two separate compaction flows in ContextWindowSelfCompactionUseCase — 'Flow 1: Soft Threshold at Done Boundary' and 'Flow 2: Hard Threshold Emergency Interrupt' — were unified into a single `performCompaction(sessionEntry, trigger: CompactionTrigger)` method.

### CompactionTrigger enum
- `DONE_BOUNDARY`: no pre-interrupt, lazy respawn post-compaction
- `EMERGENCY_INTERRUPT`: Ctrl+C + race guard pre, immediate respawn post

### Shared core steps (both triggers)
1. Reset signal (fresh deferred)
2. Send self-compaction instruction (EMERGENCY variant: interrupt-acknowledgment prefix)
3. Await AgentSignal.SelfCompacted (5-min timeout)
4. Validate PRIVATE.md
5. Git commit
6. Kill session

### Benefits
- Single well-tested code path — changes only need to happen once
- Eliminates divergence risk between the two flows
- Cleaner separation: entry-point detection vs shared compaction logic

### Affected specs
- ContextWindowSelfCompactionUseCase.md: main rewrite
- PartExecutor.md: handleEmergencyCompaction → performCompaction(EMERGENCY_INTERRUPT)
- high-level.md: context window summary updated
