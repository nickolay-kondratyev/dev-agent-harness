## FailedToExecutePlanUseCase (ref.ap.RJWVLgUGjO5zAwupNLhA0.E)

Implements the failure handling use case from doc/use-case/HealthMonitoring.md § FailedToExecutePlanUseCase Detail.

### New files
- `FailedToExecutePlanUseCase.kt` — interface + impl (4-step deterministic cleanup)
- `AllSessionsKiller.kt` — interface; `TmuxAllSessionsKiller.kt` — runs `tmux kill-server`
- `ProcessExiter.kt` — testable exit wrapper
- `ConsoleOutput.kt` — testable console output (ANSI red)
- `TicketFailureLearningUseCase.kt` — interface + NoOp stub (real impl deferred)
- 15+ BDD unit tests including ordering verification

### Design decisions
- `tmux kill-server` for session cleanup (deterministic, handles no-server-running gracefully)
- All dependencies injectable via constructor for testability
- Exhaustive `when` on PartResult sealed class (no `else` branch)
