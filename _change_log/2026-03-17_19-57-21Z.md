## What changed

Removed the /callback-shepherd/query/validate-plan endpoint and callback_shepherd.query.sh script from the spec. Plan validation now happens exclusively harness-side in convertPlanToExecutionParts — single source of truth.

## Spec files updated

- **doc/core/agent-to-server-communication-protocol.md**: Removed validate-plan endpoint, Plan Validation Query section, query script, and Two-Tier design (simplified to signal-only).
- **doc/use-case/SetupPlanUseCase/DetailedPlanningUseCase.md**: Added role-file-existence check (rule f) to harness validation. Updated failure behavior: instead of halt-immediately, logs WARN and restarts the planning loop with errors injected as planner context.
- **doc/core/ContextForAgentProvider.md**: Removed validate-plan from Planner/Plan Reviewer callback instructions and compaction-survival block.
- **doc/high-level.md**: Updated architecture paragraphs and design decisions table.
- **doc/use-case/HealthMonitoring.md**: Removed validate-plan from liveness callback list.
- **doc/schema/plan-and-current-state.md**: Updated loadsPlan field doc.
- **doc/core/PartExecutor.md**: Removed Tier 2 query paragraph.
- **doc/core/TicketShepherd.md**: Updated convertPlanToExecutionParts failure flow.

## Why

Single validation point eliminates drift between agent-side and harness-side validation. Simpler agent instructions. Solid retry path if harness validation catches a bad plan.
