## What changed
Updated `doc/core/ContextForAgentProvider.md` (ref.ap.9HksYVzl1KkR9E1L2x8Tx.E).

### Interface section
- Removed 4 methods: `assembleDoerInstructions`, `assembleReviewerInstructions`, `assemblePlannerInstructions`, `assemblePlanReviewerInstructions`
- Added `AgentRole` enum: `DOER, REVIEWER, PLANNER, PLAN_REVIEWER`
- Single method: `assembleInstructions(role: AgentRole, request: UnifiedInstructionRequest): Path`

### UnifiedInstructionRequest
- Replaces 4 separate request data classes
- Common fields at top; role-specific fields grouped with comments and defaulted to `null`/`emptyList()`

### Internal design
- Updated `assembleFromPlan` signature reference to use `UnifiedInstructionRequest`
- Updated Benefits bullet for extensibility
- Updated Doer/Reviewer table headers

## Code not touched
This is a spec-only change. Implementation (`ContextForAgentProvider.kt`, `ContextForAgentProviderImpl.kt`) still has the old 4-method shape — a follow-up implementation ticket should apply the code change.
