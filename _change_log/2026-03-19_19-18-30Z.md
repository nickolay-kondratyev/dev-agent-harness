## ShepherdServer Implementation

### New Files
- `app/src/main/kotlin/com/glassthought/shepherd/core/server/ShepherdServer.kt` — Ktor CIO server with configureApplication() pattern for testability
- `app/src/main/kotlin/com/glassthought/shepherd/core/server/SignalRequests.kt` — 6 request DTOs
- `app/src/test/kotlin/com/glassthought/shepherd/core/server/ShepherdServerTest.kt` — 23 BDD tests

### Modified Files
- `Constants.kt` — Added SERVER_PORT_ENV_VAR to AGENT_COMM
- `ShepherdValType.kt` — Added HANDSHAKE_GUID, RESULT, REASON, ROLE
- `ProtocolVocabulary.kt` — Added Signal.STARTED
- `SessionEntry.kt` — Changed lastActivityTimestamp from Instant to AtomicReference<Instant>
- `build.gradle.kts` + `libs.versions.toml` — Added ktor-server-test-host and ktor-server-status-pages

### Design Decisions
- lastActivityTimestamp changed to AtomicReference<Instant> for thread-safe mutation (matches pendingPayloadAck pattern)
- configureApplication() is public for Ktor testApplication sharing
- compareAndSet NOT used for ack-payload because PayloadId is a value class (boxing breaks reference equality)
- StatusPages catches MissingKotlinParameterException for clean 400 on malformed JSON
