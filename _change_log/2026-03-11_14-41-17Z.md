## What was done

- `Initializer.initialize()` now accepts `OutFactory` as a parameter (previously hardcoded `SimpleConsoleOutFactory`). All callers updated.
- Created `SharedAppDepIntegFactory` (object singleton): provides shared `AppDependencies` and `TestOutManager` for integration tests.
- Created `SharedAppDepSpecConfig` (data class) and `SharedAppDepDescribeSpec` (abstract base class) in `com.glassthought.chainsaw.integtest` package.
- Updated `SpawnTmuxAgentSessionUseCaseIntegTest` to extend `SharedAppDepDescribeSpec` — removes inline dependency construction.
- Updated `ai_input/memory/auto_load/4_testing_standards.md` with guidance on `SharedAppDepDescribeSpec`.

## Key implementation notes
- Body lambda receiver type is `SharedAppDepDescribeSpec.() -> Unit` (cast to `AsgardDescribeSpec.() -> Unit` for super constructor) — enables natural `appDependencies` access in test bodies.
- JVM platform clash avoided by removing redundant explicit getter methods from `SharedAppDepIntegFactory` (Kotlin property getters already serve that role).
- Anchor point `ap.20lFzpGIVAbuIXO5tUTBg.E` on `SharedAppDepDescribeSpec`.
