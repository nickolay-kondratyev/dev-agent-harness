## Changes

Spec-only change (no code changes).

- `doc/core/DirectLLM.md`: Replaced interface-per-tier design with single `DirectLLM` interface. Model selection is now a constructor parameter of the implementation, injected at wiring time via `ContextInitializer`.
- `doc/high-level.md`: Updated DirectLLM section and linked docs table.
- `doc/core/UserQuestionHandler.md`: `LlmUserQuestionHandler` references `DirectLLM` (not `DirectBudgetHighLLM`).
- `doc/core/git.md`: Removed `DirectQuickCheapLLM` mention in slug truncation rationale.

## Rationale

Premature abstraction: V1 has exactly 2 concrete uses, both harness-internal. Tier concept assumed future mid-tier additions that may never materialize. Single interface + DIP via constructor injection is simpler and equally extensible.
