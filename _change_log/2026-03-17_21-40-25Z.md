## What changed

Three spec documents updated to reflect a simplified AgentFacade interface:

### AgentInteraction.md (AgentFacade spec)
- Replaced `spawnAgent(config)` + `sendPayloadWithAck(handle, payload)` with unified `sendPayload(config, existingHandle, payload): SpawnedAgentHandle`
- The new method transparently manages session lifecycle: spawns if handle is null/dead, re-uses if alive, auto-respawns on send-keys failure
- Always creates a fresh signal deferred — resolves open question R1 (fresh deferreds)

### PartExecutor.md
- Removed `if (liveHandleExists)` branch from re-instruction pattern
- Executor always calls `sendPayload(config, handle, instructions)` regardless of session state
- Replaced "Idle Session Death → AgentCrashed" with transparent respawn via facade
- Simplified steps 1b/2b flow

### ContextWindowSelfCompactionUseCase.md
- Removed the `if/else` session existence check code block
- Collapsed "reset signal" + "spawn new session" + "send instructions" into single `sendPayload(config, null, instructions)` call
- Unified idle session death and self-compaction under same sendPayload code path

## Benefits
- Three respawn triggers (first-run, post-compaction, idle-session-death) unified in one place (AgentFacade)
- Executor is agnostic to session lifecycle — no branching on session state
- Idle session death no longer surfaces as AgentCrashed — handled transparently
- FakeAgentFacade simpler to program in tests (one method instead of two)
