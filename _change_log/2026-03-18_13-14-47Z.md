## What changed

Removed validatePresence() from the ContextWindowStateReader interface spec and all related spec sections.

### Files updated
- doc/use-case/ContextWindowSelfCompactionUseCase.md
- doc/core/PartExecutor.md

### Spec changes
- Interface now has single  method (was two:  + )
- Removed the 'context_window_slim.json Validation After Session ID Resolution' section
- Removed R2 requirement; renumbered R3-R9 → R2-R8 and updated Gate scopes
- Updated PartExecutor.md precondition comment to reflect natural failure on first read()

### Rationale
read() already throws ContextWindowStateUnavailableException for missing files — same
hard-stop behavior. First read() happens within seconds of spawn (first health check).
Single-method interface = simpler contract, no divergence risk between methods.
