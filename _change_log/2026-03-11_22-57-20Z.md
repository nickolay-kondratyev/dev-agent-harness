## Changes

- Added `handshakeGuid: HandshakeGuid` as first field in `ResumableAgentSessionId`
- `ClaudeCodeAgentSessionIdResolver` passes the GUID through instead of discarding it
- Updated 3 test assertions accordingly

## Why

SessionsState is keyed by HandshakeGuid. The GUID must travel with session identity for server routing and `current_state.json` persistence (handshake_guid field). Previously the resolver received the GUID but did not store it in the returned identity object.
