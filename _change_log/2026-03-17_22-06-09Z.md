## Change
Replaced fail-fast description with collect-all-failures spec:
- All three checks (Docker, tmux, required env vars) always run
- Each check is a pure function returning String? (null=ok, non-null=error message)
- validate() collects non-null results into List<String> and throws once at the end
- Error format shows every issue in a consolidated message

## Rationale
User in a new environment previously needed N fix-and-retry cycles to discover all missing
prerequisites. Collecting all failures up front lets the user fix everything in one go.
