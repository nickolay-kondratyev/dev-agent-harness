## What changed
- New `SessionExistenceChecker` fun interface in `tmux` package
- `TmuxSessionManager` implements `SessionExistenceChecker`
- `TmuxSession.existsChecker` field type changed from `suspend () -> Boolean` to `SessionExistenceChecker`
- `TmuxSession.exists()` calls `existsChecker.exists(name)` instead of invoking lambda
- `createSession` passes `this` instead of `{ sessionExists(sessionName) }`

## Why
Makes the dependency visible in the type hierarchy, enables clean mocking, and eliminates implicit capture-by-reference fragility. Aligns with project standard: "Be classy and use interfaces".
