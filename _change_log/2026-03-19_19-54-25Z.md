## Key Design Decisions
- `PlanningPartExecutorFactory` accepts `priorConversionErrors: List<String>` so conversion validation errors are injected into planner context on retry
- `maxConversionRetries` is a separate budget from executor iteration config (intentional deviation from spec wording for cleaner separation)
- Session cleanup delegated to PartExecutorImpl (already kills sessions on all terminal paths)
