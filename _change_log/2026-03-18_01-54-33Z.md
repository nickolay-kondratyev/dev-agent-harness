## Binex vs Shepherd Analysis

Key incompatibilities:
- **Execution model**: Binex = stateless fire-once nodes via LLM API calls; Shepherd = persistent TMUX sessions with send-keys communication
- **Iteration**: Binex = acyclic DAG with conditional branching; Shepherd = cyclic doer-reviewer feedback loop within same session
- **Language**: Python vs Kotlin/JVM
- **Communication**: DAG data flow vs TMUX send-keys + HTTP callbacks with ACK protocol

Conclusion: Do not extend binex. The effort exceeds building from scratch with no meaningful reuse.
