## What changed

Updated all `agentType` string values from `ClaudeCode` to `CLAUDE_CODE` across:

- **Workflow configs**: `config/workflows/straightforward.json`, `config/workflows/with-planning.json`
- **Instruction text**: `InstructionText.kt` (table + plan format example)
- **KDoc comments**: `SubPart.kt`, `SessionRecord.kt`
- **Tests**: `PlanCurrentStateModelTest`, `CurrentStateDerivedStatusTest`, `AiOutputStructureEnsureStructureTest`, keyword assertion tests

## Why

The `AgentType` enum defines `CLAUDE_CODE` (UPPER_SNAKE_CASE). The normalization in `SubPartConfigBuilder.parseAgentType` applies `.uppercase().replace(" ", "_")` which converts `ClaudeCode` → `CLAUDECODE` (not `CLAUDE_CODE`), causing startup failure:
```
Invalid agentType=[ClaudeCode] for sub-part=[impl]. Valid values: [CLAUDE_CODE, PI]
```
