## Changes

### New files
- `GlmConfig.kt` (ap.8BYTb6vcyAzpWavQguBrb.E) — Data class encapsulating GLM env vars with `toEnvVarExports()` method
- `ClaudeCodeAdapterSpawnIntegTest.kt` — Integration test spawning real agent via GLM

### Modified files
- `ClaudeCodeAdapter.kt` — Added optional `GlmConfig?` parameter; prepends GLM env vars to innerCommand when non-null
- `ContextInitializer.kt` — Added `forIntegTest()` factory with `glmEnabled=true`; production `standard()` does NOT enable GLM
- `SharedContextIntegFactory.kt` — Uses `forIntegTest()` instead of `standard()`
- `ClaudeCodeAdapterTest.kt` — Unit tests for GLM config injection (with/without config, quoting)

### Key decisions
- GLM env vars are double-quoted in exports to prevent shell injection
- `GlmConfig` nullability (not Environment.isTest) controls injection — simpler and more explicit
- Integration test validates: session existence, GUID-based session ID resolution, send-keys interactivity
- Bootstrap handshake (/signal/started) is indirectly verified via session ID resolution success
