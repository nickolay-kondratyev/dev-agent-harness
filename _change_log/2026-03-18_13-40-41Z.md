Updated `doc/core/agent-to-server-communication-protocol.md` (ap.wLpW8YbvqpRdxDplnN7Vh.E) to specify the new PayloadId format.

**Old format:** 21 random alphanumeric characters (e.g., `aB3xK9mR2pL7nQ4wY6jHt`)

**New format:** `{handshakeGuid_short_8chars}-{sequenceN}` (e.g., `a1b2c3d4-3` for 3rd payload in that session)

**Why:** Simpler (counter vs random generation) and more observable (self-correlating ID reveals session + sequence, gaps are detectable, deterministic for tests).

Spec sections updated: PayloadId format description, Kotlin snippet, XML wrapper example, signal payload example, callback script comment.
