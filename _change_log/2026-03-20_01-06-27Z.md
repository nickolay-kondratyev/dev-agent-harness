## Changes
- Created `TicketStatusUpdaterImpl` — production impl that calls `ticket close <ticketId>` via ProcessRunner
- Added `TicketStatusUpdaterFactory` fun interface for deferred construction (ticketId only known at create-time)
- Changed `TicketShepherdCreatorImpl` constructor from `ticketStatusUpdater` to `ticketStatusUpdaterFactory` with production default
- Added `ShepherdValType.TICKET_ID` for structured logging
- Unit tests: data-driven happy path + failure propagation test

## Design Decisions
- Factory pattern matches existing `AllSessionsKillerFactory` convention
- Uses `ticket` executable (not `tk` alias) for PATH reliability
- ProcessRunner for testable subprocess execution
