## What Changed

- **ContextForAgentProvider.kt**: Added AgentRole enum, UnifiedInstructionRequest data class (spec-exact), single assembleInstructions method. RoleCatalogEntry extracted from InstructionSections to top-level (fixes layering violation).
- **ContextForAgentProviderImpl.kt**: Single public method dispatches via exhaustive when(role) to private builders. Added partName to debug log for execution agents.
- **InstructionSections.kt**: Removed nested RoleCatalogEntry (moved to ContextForAgentProvider.kt).
- **ContextTestFixtures.kt + 4 test files**: All call sites updated to use UnifiedInstructionRequest with AgentRole.
- **ContextForAgentProviderAssemblyTest.kt**: 4 new requireNotNull guard tests.

## Why
Spec-alignment. The spec (doc/core/ContextForAgentProvider.md) defined the unified interface; the impl had diverged to 4 methods + 4 request types with heavy field overlap. Adding a new agent role now requires only a new AgentRole variant + plan list, not a new method/request/builder triple.

## Bugs found in review
- RoleCatalogEntry was nested in InstructionSections (rendering layer) but referenced from UnifiedInstructionRequest (data model) — inverted dependency. Fixed by elevating to top-level.
