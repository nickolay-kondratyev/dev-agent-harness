## What changed
- `doc/core/git.md`: Removed `onPartDone` from hook table, eliminated `CommitPerPart` strategy row, simplified to single-hook interface description. Updated failure handling and git operations summary tables.
- `doc/core/TicketShepherd.md`: Removed `GitCommitStrategy.onPartDone` calls from planning step 3d and execution step 4d. Updated responsibilities to clarify git commits are owned by executor via `onSubPartDone`.

## Why
Ticket decision: V1 needs maximum commit granularity to audit agent communication (PUBLIC.md overrides). `onPartDone` was redundant when `CommitPerSubPart` already commits after each sub-part. Removing it simplifies the interface from 2 methods to 1.

## Tradeoff
No part-level-only commit strategy available. Acceptable for V1 — can be re-added later if needed.
