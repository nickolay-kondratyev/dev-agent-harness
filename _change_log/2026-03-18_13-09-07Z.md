## Summary

Researched whether OpenClaw (openclaw.ai) and DevClaw (github.com/laurentenhoor/devclaw) duplicate or replace what TICKET_SHEPHERD aims to build.

## Findings

- **OpenClaw**: self-hosted chat gateway (WhatsApp/Telegram/Discord → AI agents). Not an agent orchestration harness.
- **Lobster tool**: deterministic pipeline runner with human approval checkpoints. Not multi-agent orchestration.
- **DevClaw**: OpenClaw plugin that turns it into a GitHub-issue-driven dev pipeline manager (3-tier: orchestrator/workers/scheduler). State lives in GitHub/GitLab labels.

## Key Result

OpenClaw/DevClaw do NOT replace TICKET_SHEPHERD. They lack:
- OS-level process isolation with HandshakeGuid protocol
- HTTP callback communication + TMUX send-keys delivery with ACK
- Context window self-compaction with threshold monitoring
- Failure learning / cross-try knowledge transfer
- Structured plan-based orchestration (SetupPlan → DetailedPlan → PartExecutor)
- Health monitoring with ping/crash detection
- NonInteractive agent runner for utility tasks

Overlap is only conceptual (both route work to AI agents). TICKET_SHEPHERD addresses a more focused and technically deeper problem.
