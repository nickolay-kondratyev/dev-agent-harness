## What Changed

Updated spec in `doc/core/git.md` (ref.ap.BvNCIzjdHS2iAP4gAQZQf.E):

- **Removed** the entire 'Model Version Resolution' section (`config/model-versions.json`, startup validation, lookup logic)
- **Updated** commit author format from `CC_sonnet-v4.6_WITH-username` → `CC_sonnet_WITH-username`
- **Removed** `VERSION_OF_MODEL` component from author table
- **Added** Why-not comment explaining YAGNI rationale inline

## Why

config/model-versions.json required manual upkeep, created config-drift risk (stale version in config while agent runs newer), and was a startup failure mode. V1 has no analysis tooling to consume version granularity — YAGNI.
