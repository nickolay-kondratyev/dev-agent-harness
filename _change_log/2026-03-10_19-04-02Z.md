## Problem
Gradle resolves maven coordinates at configuration time, before any task runs.
The previous `dependsOn(:ensureAsgardInMavenLocal)` in compileKotlin could never
self-heal missing asgard libs — the build failed at configuration before the task executed.

## Fix
Added `_prepare_pre_build.sh` (ap.gtpABfFlF4RE1SITt7k1P.E) with:
- `_prepare_asgard_dependencies`: fast file-stat check, slow-path publish via `publishAsgardToMavenLocal`
- Auto-sets THORG_ROOT internally
- Wired into `test.sh` and `test_with_integ.sh`

## Changes
- `_prepare_pre_build.sh` — new script
- `test.sh` / `test_with_integ.sh` — source and call `_prepare_asgard_dependencies`
- `app/build.gradle.kts` — removed ineffective `dependsOn`, added explanatory comment
- `build.gradle.kts` — updated `ensureAsgardInMavenLocal` doc (now manual-use only)
- `ai_input/memory/auto_load/0_env-requirement.md` — updated docs
