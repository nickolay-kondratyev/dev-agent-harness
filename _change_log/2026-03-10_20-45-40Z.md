## What was done

- Added `tasksJson` task in `build.gradle.kts`: writes all project tasks to `build/tasks-json/tasks.json`. Input fingerprinting on all build files + settings.gradle.kts gives proper Gradle UP-TO-DATE caching — fast repeated calls (no re-execution when nothing changed).
- Created `gradle_tasks_jsonl_cached.sh`: runs `./gradlew tasksJson` (fast if UP-TO-DATE) then streams the JSON array as JSONL.
- Created `gradle_run.sh`: fzf-based interactive task picker that calls `gradle_tasks_jsonl_cached.sh` and runs the selected task via `eai2` (or falls back to `./gradlew`).

## Why Gradle-level caching

Shell-level memoization (e.g. `memoize_by_pwd`) cannot detect when tasks actually change. Gradle UP-TO-DATE checking uses content hashing of build files as inputs — cache is automatically invalidated when any build file changes.
