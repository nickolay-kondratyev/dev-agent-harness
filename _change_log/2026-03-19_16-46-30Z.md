## Details

7 new sealed subtypes added to `InstructionSection` for execution (doer/reviewer) and planner agents:

### Execution-specific
- **PlanMd** — reads PLAN.md from ExecutionContext, null=skip, missing=fail-hard
- **PriorPublicMd** — renders prior completed PUBLIC.md files per visibility rules
- **IterationFeedback** — reviewer feedback + pushback guidance wrapped in compaction-survival tags
- **InlineFileContentSection(heading, path?)** — generic file content renderer, null=skip, non-null+missing=fail-hard

### Planner-specific
- **RoleCatalog** — delegates to InstructionRenderers.roleCatalog()
- **AvailableAgentTypes** — static V1 agent types text
- **PlanFormatInstructions** — static JSON schema for plan_flow.json

### Testing
- ~400 lines of BDD tests covering all subtypes
- Fail-hard tests for PlanMd, PriorPublicMd, IterationFeedback, InlineFileContentSection
- Negative test for PriorPublicMd visibility rules
- All tests pass via ./test.sh
