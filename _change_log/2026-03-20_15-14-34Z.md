## Root Causes Fixed

1. **TICKET_SHEPHERD_SERVER_PORT not exported** — ClaudeCodeAdapter now exports this env var into each agent tmux session so callback scripts know which port to POST to.

2. **callback_shepherd.signal.sh not on PATH** — ClaudeCodeAdapter now adds the callback scripts directory to PATH in the tmux session.

3. **Bootstrap message too vague** — SubPartConfigBuilder.BOOTSTRAP_MESSAGE now explicitly instructs agents to call `callback_shepherd.signal.sh started` immediately (was just "Waiting for instructions.").

4. **ContextInitializerImpl wiring** — Reads server port from env var and extracts callback scripts from classpath resources at runtime. Supports overrides for integration test compatibility.

## Integration Test Fix

ServerPortInjectingAdapter updated to **replace** sentinel exports instead of prepending, preventing sentinel values from overwriting real values in bash sequential export processing.
