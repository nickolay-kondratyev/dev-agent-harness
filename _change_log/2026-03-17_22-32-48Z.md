Updated `doc/use-case/AutoRecoveryByAgentUseCase.md` (spec only, no code changes).

## Changes

- **RecoveryResult → RecoveryOutcome**: sealed class now has three explicit exits:
  - `Resolved` — recovery succeeded, caller retries original operation once
  - `Unresolvable` — all `maxRecoveryAttempts` exhausted, use case emits FAIL with reason `"recovery_exhausted"`
  - `NeedsEscalation(reason)` — agent explicitly signals human intervention needed (returned immediately)

- **maxRecoveryAttempts: Int = 2**: constructor config parameter; use case owns the internal retry loop — callers no longer implement retry logic

- **Flow**: updated to show bounded internal loop (spawn → fail → retry up to N times → Unresolvable)

- **Caller Protocol**: simplified — callers just pattern-match on RecoveryOutcome; no retry logic on caller side
