## What changed
- `TmuxCommunicator` is now an interface; impl is `TmuxCommunicatorImpl`
- New `TmuxSession` class encapsulates session name + communicator + existence checker
  - `sendKeys(text)` — no session arg needed
  - `sendRawKeys(keys)` — no session arg needed
  - `exists()` — replaces `TmuxSessionManager.sessionExists(name: String)`
- `TmuxSessionManager.createSession()` returns `TmuxSession` (was `TmuxSessionName`)
- `TmuxSessionManager.killSession()` takes `TmuxSession` (was `TmuxSessionName`)
- `sessionExists()` made private on manager
- `App.kt` wired with `TmuxCommunicatorImpl`, uses `session.sendKeys()` directly
- Tests updated to new API; duplicate test case removed

## Design notes
- Lambda `existsChecker: suspend () -> Boolean` in `TmuxSession` avoids circular dependency with `TmuxSessionManager`
- Anchor point `ap.3BCYPiR792a2B8I9ZONDwmvN.E` preserved on `TmuxCommunicatorImpl`
