## What Changed

### Interface (ContextForAgentProvider.kt)
- Deleted: `assembleExecutionAgentInstructions` + `ExecutionAgentInstructionRequest`
- Added: `assembleDoerInstructions(DoerInstructionRequest)` + `assembleReviewerInstructions(ReviewerInstructionRequest)`
- `isReviewer: Boolean` eliminated — type system enforces role at compile time
- Field names explicit: `reviewerPublicMdPath` (doer), `doerPublicMdPath` (reviewer)
- `feedbackDir` only on ReviewerInstructionRequest (was irrelevant to doer)

### Implementation (ContextForAgentProviderImpl.kt)
- `buildDoerSections()` — linear template, no role-dispatching
- `buildReviewerSections()` — linear template, no role-dispatching
- Iteration > 1 conditionals kept as simple `if` inside each template (80/20)
- Planner/PlanReviewer methods unchanged (already role-specific)

### Spec (ContextForAgentProvider.md)
- Split combined Doer/Reviewer concatenation table into separate Doer and Reviewer tables
- Updated interface section to show 4 methods

## Tradeoff
Sections 1-6 (role def, part context, ticket, shared context, plan, prior outputs) appear in both doer and reviewer templates. This duplication is in section ordering, not content (content lives in InstructionSections and private helpers). Each template is ~20 lines — extracting common prefix would re-introduce the indirection we eliminated.
