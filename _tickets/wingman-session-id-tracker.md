Implement the Wingman interface and ClaudeCodeWingman for discovering Claude Code session IDs via GUID handshake.

## Scope
- Create `Wingman` interface: `suspend fun resolveSessionId(guid: String): String`
- Create `ClaudeCodeWingman` implementation
- Scan `~/.claude/projects/.../*.jsonl` files for content containing the GUID
- Extract session ID from matched filename (e.g., `77d5b7ea-cf04-453b-8867-162404763e18.jsonl` → session ID)
- Package: `com.glassthought.chainsaw.core.wingman`

## Key Decisions
- GUID is generated by the harness and sent to the agent as the first message in a TMUX session
- ClaudeCodeWingman searches JSONL files by reading content and matching the GUID string
- Fail-fast if no matching file found (throw descriptive exception)
- Fail-fast if multiple matches found (ambiguous state)
- No new dependencies needed — can use existing `org.json` or plain string matching

## Testing
- Unit tests using temp directories with sample JSONL files
- Test: GUID found in exactly one file → returns session ID from filename
- Test: GUID not found → throws descriptive exception
- Test: GUID found in multiple files → throws ambiguous match exception
