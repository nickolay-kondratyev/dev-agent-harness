## Part Completion Guard

New `PartCompletionGuard` class (SRP) injected into `PartExecutorImpl` via `PartExecutorDeps`.

### Behavior
- After reviewer PASS + PUBLIC.md validation: scans `__feedback/pending/` for `critical__*` and `important__*` files
- If found → `PartResult.AgentCrashed` (no retry — broken agent)
- If only `optional__*` remain → moves to `addressed/` (implicitly accepted as skipped)
- Empty/non-existent pending → passes

### Files
- `PartCompletionGuard.kt` — guard logic with sealed `GuardResult`
- `PartExecutorImpl.kt` — wired into PASS branch via `validateCompletionGuardOrCrash()`
- `PartCompletionGuardTest.kt` — 13 unit tests
- `PartExecutorImplTest.kt` — 5 integration tests
