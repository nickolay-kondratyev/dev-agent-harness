## Changes

Simplified the SubPartStatus state machine by removing the only back-transition (COMPLETED→IN_PROGRESS).

### Files Updated
- **doc/schema/plan-and-current-state.md**: Removed COMPLETED→IN_PROGRESS transition, ResumeForIteration sealed class entry, and validateCanResumeForIteration() validator. Updated Complete KDoc. Updated JSON examples to show doer as IN_PROGRESS during iteration. Updated persistence timing table.
- **doc/core/PartExecutor.md**: Updated flow steps 2 and 3. Updated Status Mutation Protocol table.

### Key Design Change
Doer stays IN_PROGRESS throughout all iteration cycles. Only marked COMPLETED when the entire part completes (reviewer PASS or doer-only done). Executor marks both reviewer and doer COMPLETED simultaneously on reviewer PASS.

### What Was Removed
- SubPartStateTransition.ResumeForIteration
- SubPartStatus.validateCanResumeForIteration()
- COMPLETED→IN_PROGRESS transition row from all tables and transition diagrams
