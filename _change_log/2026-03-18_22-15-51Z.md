## Changes
- Added `org.sonarqube` plugin v5.1.0.4882 to `gradle/libs.versions.toml`
- Added `sonar {}` config block in root `build.gradle.kts` (org: nickolay-kondratyev, project: nickolay-kondratyev_dev-agent-harness)
- Created `run_sonar.sh` with:
  - SONAR_TOKEN validation (fail-fast)
  - Gradle analysis (`--no-configuration-cache`)
  - API report fetch with `fetch_sonar_api` helper (curl -sf + JSON validation)
  - Combined JSON report: quality gate + metrics + issues → `_reports/sonar_report.json`
- Added `_reports/` to `.gitignore`

## Design Decisions
- Token uses `?: ""` in Gradle config (not `?: error()`) — error() would break ALL gradle commands when SONAR_TOKEN is unset
- Uses `Authorization: Bearer` header (SonarCloud recommended over legacy basic auth)
- Sonar task is NOT wired to build/test — explicit invocation only
