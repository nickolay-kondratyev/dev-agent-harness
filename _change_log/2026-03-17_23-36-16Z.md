Decision: KEEP optional severity, not eliminate it.

## Changes

### 1. SKIPPED resolution marker
- New `## Resolution: SKIPPED` marker valid only for `optional__` prefixed feedback files
- Previously doer wrote ADDRESSED when skipping, which was misleading
- SKIPPED = "I saw this and chose not to act" (explicit, auditable)
- Harness moves SKIPPED files to `addressed/`

### 2. Documented importance of optional feedback
- Added "Why optional feedback is kept" rationale in D4 of granular-feedback-loop.md
- Optional occupies the middle ground between forced (important__) and lost (PUBLIC.md only)
