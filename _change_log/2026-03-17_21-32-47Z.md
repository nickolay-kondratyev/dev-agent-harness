## What was done

Created `HarnessTimeoutConfig` data class at `com.glassthought.shepherd.core.data` centralizing all harness timeout and threshold constants:

- **Health monitoring:** `startupAckTimeout` (3 min), `healthCheckInterval` (5 min), `noActivityTimeout` (30 min), `pingTimeout` (3 min)
- **Payload ACK:** `payloadAckTimeout` (3 min), `payloadAckRetries` (3)
- **Self-compaction:** `selfCompactionTimeout` (5 min), `contextWindowSoftThresholdPct` (35), `contextWindowHardThresholdPct` (20)

Added `HarnessTimeoutConfig.forTests()` with fast sub-second timeouts to keep unit tests snappy.

Wired into `ShepherdContext.timeoutConfig` (defaults to `HarnessTimeoutConfig.defaults()`).

Updated spec docs (HealthMonitoring.md, ContextWindowSelfCompactionUseCase.md, PartExecutor.md) to reference the new data class fields.

Added `HarnessTimeoutConfigTest` covering all production defaults and `forTests()` behavior.
