Removed `healthCheckInterval` (default 5 min) from `HarnessTimeoutConfig` spec.

Simplified the health-aware await loop pseudocode:
- `lastHealthCheck` variable removed (both the pre-loop init and the in-loop update)
- `awaitSignalWithTimeout(healthCheckInterval)` → `awaitSignalWithTimeout(1.second)`

Detection latency improves from worst-case `normalActivity + healthCheckInterval` = 35 min to exactly `normalActivity` = 30 min. One cadence instead of two.

Files changed:
- `doc/use-case/HealthMonitoring.md` — removed healthCheckInterval row from All Health Parameters table
- `doc/core/PartExecutor.md` — simplified Loop Structure pseudocode
