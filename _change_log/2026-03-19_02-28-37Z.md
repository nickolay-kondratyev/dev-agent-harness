## What was implemented

### WorkflowDefinition (ap.b4i1YCm3AvwEySAjDRoJg.E)
- Data class with `parts` (straightforward) or `planningParts` + `executionPhasesFrom` (with-planning)
- Init-block validation: mutual exclusivity, phase constraints, executionPhasesFrom rejection on straightforward

### WorkflowParser (ap.kz1nvt0wyd2UpWsCGmm7Y.E)
- Interface + WorkflowParserImpl following TicketParser pattern
- Loads `config/workflows/<name>.json`, validates phases match workflow type
- Fail-fast on missing file, malformed JSON, invalid phases
