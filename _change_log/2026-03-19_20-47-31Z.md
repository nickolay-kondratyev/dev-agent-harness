## Changes
- **ContextForAgentProvider.kt**: New `DoerFeedbackItemRequest` sealed subtype with `feedbackItem: InstructionSection.FeedbackItem` field
- **ContextForAgentProviderImpl.kt**: New when-branch + `buildDoerFeedbackItemPlan()` method (FeedbackItem instead of IterationFeedback)
- **InstructionSection.kt**: Updated 3 when-expressions for exhaustiveness with new sealed subtype
- **InnerFeedbackLoop.kt**: Updated `buildFeedbackItemRequest()` signature and `processFeedbackItem()` to read/pass feedback data
- **Tests**: 15+ new test cases covering assembly, ordering, request building; 2 existing tests updated for new flow

## Design Decision
New sealed subtype (not nullable field on DoerRequest) — follows SRP, leverages compiler exhaustiveness, keeps DoerRequest clean.
