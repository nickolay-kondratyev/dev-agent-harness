## What changed

Rewrote the UnifiedInstructionRequest design in doc/core/ContextForAgentProvider.md.

### Before
Single flat `UnifiedInstructionRequest` data class with ~15 nullable fields, discriminated by a separate `role: AgentRole` parameter passed to `assembleInstructions`. Invalid field combinations (e.g. passing reviewer fields to a doer) were runtime bugs.

### After
Sealed class hierarchy `AgentInstructionRequest` with role-specific subtypes:
- `ExecutionRequest.DoerRequest` — doer-specific fields, null only for iteration-1 optional
- `ExecutionRequest.ReviewerRequest` — previously nullable `doerPublicMdPath` and `feedbackDir` now non-nullable
- `PlannerRequest` — previously nullable `planJsonOutputPath` / `planMdOutputPath` now non-nullable
- `PlanReviewerRequest` — previously nullable plan content / planner public md now non-nullable

`assembleInstructions(role, request)` → `assembleInstructions(request)`. Internal `when` dispatch is compile-time exhaustive.

Remaining nulls are semantically meaningful optionals (absent on iteration 1 or no-planning workflows).
