Currently the harness maintains TWO representations of the plan:
1. `plan.json` (machine-readable, authoritative)
2. `shared/plan/PLAN.md` (human-readable, generated by harness from plan.json)

The harness has text-formatting logic to generate PLAN.md from plan.json and then both are injected into agent context (see doc/core/ContextForAgentProvider.md section 4 for Doer and Reviewer). This dual representation creates complexity and a risk of divergence between the two.

Proposed simplification:
- Eliminate PLAN.md generation entirely
- Pass plan.json directly to agents in context (agents are excellent at parsing structured JSON)
- Remove text-formatting logic from harness
- Update doc/schema/ai-out-directory.md to remove shared/plan/PLAN.md
- Update doc/core/ContextForAgentProvider.md to reference plan.json directly

Robustness gains:
- Single source of truth — no risk of PLAN.md diverging from plan.json
- Eliminates a code path that transforms data unnecessarily
- Simpler context assembly in ContextForAgentProviderImpl

- doc/core/ContextForAgentProvider.md (Doer/Reviewer context section 4)
- doc/schema/ai-out-directory.md

Relevant code:
