## What changed

Added `FrontmatterFields` class to `YamlFrontmatterParser.kt` with two methods:
- `require(key)` — returns the value or throws `IllegalArgumentException` with a consistent message
- `optional(key)` — returns the value or `null`

An optional `sourceContext` parameter (e.g. filename) is included in error messages when provided.

## Files changed
- `YamlFrontmatterParser.kt` — new `FrontmatterFields` class added alongside existing types
- `TicketParser.kt` — replaced 3 raw `yamlFields[...]` lookups with `FrontmatterFields`
- `RoleCatalogLoader.kt` — replaced 2 raw `yamlFields[...]` lookups with `FrontmatterFields(sourceContext = filename)`

## Tests
All existing `TicketParserTest` and `RoleCatalogLoaderTest` cases pass, including the assertion that error messages contain the role filename.
