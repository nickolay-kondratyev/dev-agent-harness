## Changes

- Removed `loadsPlan` from SubPart Fields table in plan-and-current-state.md
- Removed `loadsPlan: true` from plan_flow.json JSON examples
- Removed validation step 'at least one sub-part has loadsPlan: true' from DetailedPlanningUseCase.md
- Updated ContextForAgentProvider.md: PlanMd section, Planner instructions row 5 & 8
- Updated ai-out-directory.md: PLAN.md description and plan_flow.json vs PLAN.md comparison table

## Why
PLAN.md is written FOR implementation agents — there is no reason to hide it from any doer sub-part.
Eliminates a planner-controlled field that could be misconfigured, and removes the associated validation.
