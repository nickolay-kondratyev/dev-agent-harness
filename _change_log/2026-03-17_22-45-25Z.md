## What Changed

### doc/schema/plan-and-current-state.md
- Added `SubPartStateTransition` sealed class (ap.EHY557yZ39aJ0lV00gPGF.E) covering all 5 legal transitions: Spawn, Complete, Fail, IterateContinue, ResumeForIteration
- KDoc on each entry IS the state machine diagram
- Added 3 validator functions:
  - `SubPartStatus.transitionTo(signal: AgentSignal)` — maps Done/FailWorkflow/Crashed to transitions
  - `SubPartStatus.validateCanSpawn()` — guards NOT_STARTED pre-condition
  - `SubPartStatus.validateCanResumeForIteration()` — guards COMPLETED pre-condition for doer resume
- Added "Adding New Transitions" guide (purely additive, compiler-enforced)

### doc/core/PartExecutor.md
- Added "Status Mutation Protocol" section: no SubPart status update without validated SubPartStateTransition
- Table mapping each transition to its validator call

## Why
Previous spec had transitions described in prose only. The SubPartStateTransition enum makes the state machine auditable in one place, lets the compiler enforce exhaustiveness on new transitions, and makes invalid transitions impossible to silently accept.
