Updated doc/core/git.md (GitOperationFailureUseCase section) to describe a new fast-path that runs before AutoRecoveryByAgentUseCase.

When git fails with an error containing 'index.lock' or 'unable to lock' AND .git/index.lock exists, the harness deletes the lock and retries immediately. This avoids spawning an LLM agent for the most common git failure in CI/agent environments.

Only falls through to the agent recovery path if the fast-path does not apply or if the retry after lock removal also fails.
