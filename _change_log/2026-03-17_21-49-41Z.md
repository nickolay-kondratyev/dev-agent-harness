## What changed

**Scope: spec only (doc/core/) — no implementation files changed.**

### Problem solved
PartExecutor.md specified that the executor must (1) create a fresh CompletableDeferred, (2) re-register SessionEntry before each payload delivery. Forgetting step 1 on re-instruction is a silent-hang bug. The two steps had to be done in the right order every time — fragile.

### Solution
`AgentFacade.sendPayloadAndAwaitSignal(handle, payload): AgentSignal` is the single entry point for every payload + signal cycle. It internally:
1. Creates fresh CompletableDeferred
2. Re-registers SessionEntry (same GUID, new deferred)
3. Sends payload with ACK (3× retry)
4. Runs health-aware await loop (ap.QCjutDexa2UBDaKB3jTcF.E)
5. Returns AgentSignal

Executor just calls `sendPayloadAndAwaitSignal` and receives the signal back. Impossible to forget deferred reset.
