## Key Design
- Fire-and-forget: POST to `/callback-shepherd/signal/<action>`, check for 200, ignore body
- Retry policy per spec ref.ap.yzc3Q5TEh2EYCN03J7ZuL.E: transient (conn refused, 5xx) → retry; non-transient (400, 404) → fail immediately
- All JSON string values go through `json_escape()` including handshakeGuid and payloadId
- curl stderr captured to temp file and included in WARN/ERROR messages for debuggability
- Tests use mock curl scripts on exec-safe tmpdir (avoids /dev/shm noexec sandbox issue)
